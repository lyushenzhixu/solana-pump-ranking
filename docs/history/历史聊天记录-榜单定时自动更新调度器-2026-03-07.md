## 1. 需求：榜单每隔 5 分钟自动更新

**用户需求**  
- 希望榜单能够每隔 5 分钟自动更新，无需手动点击按钮触发。

**背景**  
- 此前榜单更新仅支持手动触发：`npm run pump-ranking` / `npm run zhilabs-ranking` 命令行方式，或前端页面点击「更新 Pump 榜单」/「更新 zhilabs 精选」按钮。  
- 曾在 2026-03-06 加入过 15s 前端轮询，但因消耗 AVE API 额度被用户要求关闭（见 `历史聊天记录-榜单页更新按钮与数据联动-2026-03-06.md` 第 4 节）。  
- 当前已替换为自研数据源（DexScreener + GeckoTerminal + Jupiter + GoPlus），无 API 额度限制，可放心定时更新。

## 2. 实现方案

### 后端调度器（`src/server.js`）

- **定时器**：使用 `setInterval` 实现，服务启动后调用 `startScheduler()`。  
- **间隔**：通过环境变量 `AUTO_UPDATE_INTERVAL_MIN` 配置，默认 5 分钟，最小 1 分钟。  
- **首次触发**：服务启动 3 秒后立即执行一次更新，确保启动后数据为最新。  
- **执行逻辑**（`runScheduledUpdate()`）：  
  1. 依次执行 `updatePumpRanking()` → `updateZhilabsRanking()`（非并行，避免 API 过载）。  
  2. 每个榜单更新前检查 `updateRunning` 锁，若手动更新进行中则跳过该榜单。  
  3. 若上一轮定时任务仍在执行（`scheduler.running === true`），整轮跳过。  
  4. 更新完成后记录 `scheduler.lastRun` 和 `scheduler.lastResult`。  
- **状态对象**：`scheduler { enabled, intervalMs, lastRun, lastResult, running, timer }`。

### 状态接口

- **`GET /api/scheduler/status`**：返回调度器当前状态 JSON。  
  ```json
  {
    "enabled": true,
    "intervalMs": 300000,
    "intervalMin": 5,
    "running": false,
    "lastRun": "2026-03-07T09:55:39.654Z",
    "lastResult": {
      "pump": { "ok": true, "count": 19 },
      "zhilabs": { "ok": true, "count": 10 },
      "startedAt": "...",
      "durationMs": 65665
    }
  }
  ```

### 前端调度状态栏

- 在榜单页 `<h1>` 与 Tab 之间新增 `.scheduler-bar` 状态栏：  
  - **圆点指示器**：绿色（空闲）/ 橙色脉冲（更新中）。  
  - **间隔显示**：「每 X 分钟」或「更新中…」。  
  - **倒计时**：`MM:SS` 格式实时递减，归零后重新拉取状态。  
  - **上次结果摘要**：「Pump 19条 · zhilabs 10条 · 65665ms」。  
- 前端每 15 秒轮询 `/api/scheduler/status` 同步状态；倒计时每秒更新。  
- 当检测到 `running === true` 时，自动刷新当前 Tab 数据。

## 3. 互斥与安全

| 场景 | 行为 |
|------|------|
| 手动更新进行中 → 定时触发 | 定时任务跳过该榜单，不重复执行 |
| 定时任务进行中 → 手动触发 | 手动更新返回 409「更新中，请稍后再试」 |
| 上一轮定时未完成 → 新一轮定时触发 | 整轮跳过，日志输出「上一轮仍在执行，跳过」 |

## 5. 配置与使用

- **默认行为**：`npm start` 即可，服务启动后自动每 5 分钟更新两个榜单。  
- **自定义间隔**：在 `.env` 中设置 `AUTO_UPDATE_INTERVAL_MIN=10`（单位：分钟）。  
- **查看状态**：访问 `/ranking` 页面顶部状态栏，或调用 `GET /api/scheduler/status`。  
- **手动更新**：仍可通过页面按钮或 `POST /api/update?type=pump|zhilabs` 触发。
