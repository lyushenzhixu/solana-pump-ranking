## 1. 问题：点击榜单刷新后，再次进入网站又变回旧数据

**用户反馈**  
- 点击榜单刷新后代币数据会更新，但下次再进入网站时，代币信息又显示为「最开始的一版」。

**原因**  
- Pump 榜单更新使用 `upsert(rows, { onConflict: 'token' })`，只做插入/更新，**不删除表中已有旧行**。  
- 每次从 AVE 拉取的是「当前符合条件的 top 20」，写入后表内 = 旧行 + 新 20 条。  
- 接口按 `tx_volume_u_24h` 排序取前 20：若旧行交易量更高，再次进入时查到的前 20 仍是旧代币，看起来像「又变回旧版」。

**实现**  
- 在 `scripts/fetch-pump-ranking.js` 中，在写入本批数据前**先清空表**再插入：  
  - 使用 `supabase.from('solana_pump_ranking').delete().gte('id', 0)` 清空表；  
  - 再执行原有 `upsert(rows, { onConflict: 'token' })`。  
- 保证表中只保留本批次 20 条，再次进入网站时展示的即为本次更新结果。  
- zhilabs 精选未改（名单来自 ca.md，upsert 语义合理）。

## 2. 问题：.env 已配置 AVE API Key，前端点更新仍提示缺少

**用户反馈**  
- 已在 .env 中配置 AVE 的 API key，但点击网页「更新榜单」仍提示缺少 AVE_API_KEY。

**原因**  
- `dotenv/config` 按**进程当前工作目录**查找 `.env`。从非项目根目录启动（如 IDE 调试、其他目录执行）时可能找不到 `.env`。  
- ES 模块中，脚本在 `import` 时就会读取 `process.env.AVE_API_KEY`，若此时尚未从项目根加载 `.env`，会得到空值。

**实现**  
- **从项目根固定加载 .env**  
  - 新增 `src/load-env.js`，用 `path.join(__dirname, '..', '.env')` 从项目根加载 `.env`。  
  - 在 `src/server.js` 最前面增加 `import './load-env.js'`，确保在拉榜单等脚本之前执行，脚本内的 `process.env.AVE_API_KEY` 能正确读到。  
- **启动时检查**  
  - 若未配置 AVE_API_KEY，在控制台输出警告，提示在 `.env` 或部署环境变量中配置并重启服务。  
- **接口错误提示**  
  - 当因缺少 AVE API Key 报错时，在返回的 error 中追加说明：确认 `.env` 已设置并重启；若为线上部署，需在平台环境变量中配置 AVE_API_KEY。

**使用说明**  
- 本地：确保项目根目录 `.env` 中有 `AVE_API_KEY=你的key`，修改后需**重启服务**。  
- 线上（如 Railway）：在平台的环境变量中配置 `AVE_API_KEY`，`.env` 不会随代码部署。
