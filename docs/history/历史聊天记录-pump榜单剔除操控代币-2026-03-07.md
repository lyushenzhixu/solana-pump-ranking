# 历史聊天记录：Pump 榜单剔除操控代币（2026-03-07）

本记录保存「剔除 pump 榜单中疑似人为操控代币」的分析、方案设计与实现过程。

## 1. 问题描述

Pump ranking 榜单中出现一批可疑代币，特征如下：

- **无图片**：`logo_url` 为空
- **名称拼凑**：如 `momentum DEX BILLIONS_NTWK_COIN`、`L1 YELLOW_COIN raw`、`DAtalayer INKONCHAIN_COIN ASSet`，由随机加密术语 + `_COIN`/`_NTWK` 组合而成
- **insider 指数极高**：AVE token 详情返回的 `insider_wallet_rate` 高达 80+（即 8000%+），远超正常代币的 0～0.2

这些代币来自 `GET /v2/ranks?topic=solana` 榜单，在 190 条结果中有 38 条无图片，全部为此类操控代币。

## 2. 分析过程

### 2.1 数据来源排查

三个数据源中，`pump_in_new` 和 `pump_in_hot`（platform API）的代币基本都有图片；问题代币集中在 `ranks?topic=solana` 中。

### 2.2 AVE API 风控字段调研

调用 AVE token 详情 `GET /v2/tokens/{address}-solana`，在 `data.token` 中发现以下风控字段：

| 字段 | 说明 |
|------|------|
| `insider_wallet_rate` | insider 钱包占比（原始值，1.0 = 100%） |
| `rug_risk_rate` | Rug 风险评分（0-100） |
| `rag_risk_rate` | Rag 风险评分（0-100） |
| `ave_risk_level` | AVE 风险等级（0=低） |
| `risk_score` | 风险评分 |

### 2.3 操控代币实测数据

| 代币 | 名称 | 市值 | insider_wallet_rate | logo |
|------|------|------|---------------------|------|
| BILl | momentum DEX BILLIONS_NTWK_COIN | 871K | **86.27**（8627%） | 无 |
| YELLow | L1 YELLOW_COIN raw | 910K | **87.40**（8740%） | 无 |
| gf | FLash INDEXEr GOLDFINGERRWA_COIN | 794K | **91.24**（9124%） | 无 |
| INk | DAtalayer INKONCHAIN_COIN ASSet | 932K | **90.01**（9001%） | 无 |
| Bp | MODUlar BACKPACK_COIN PLATform | 1535K | **91.68**（9168%） | 无 |

正常代币（如热门 pump 代币）的 `insider_wallet_rate` 通常在 0～0.21 之间。

### 2.4 方案选择

- **方案 A**：正则匹配名称模式（`_COIN`、`_NTWK` 等）→ 误杀风险高，且操控者可随时换名
- **方案 B**：AVE Contract Risk API `/v2/contracts/{addr}-solana` → 额外 API 调用，且 `ave_risk_level` 对这些代币返回 0
- **方案 C（采用）**：`logo_url` + `insider_wallet_rate` 双重过滤 → 零额外 API 调用（复用 token 详情请求），覆盖率 100%

## 3. 实现方案

### 3.1 新增常量

```javascript
const EXCLUDE_NO_LOGO = true;       // 排除无图片代币
const MAX_INSIDER_RATE = 0.50;      // insider_wallet_rate 阈值（0.5 = 50%）
```

### 3.2 修改点

| 文件 | 修改内容 |
|------|----------|
| `scripts/fetch-pump-ranking.js` | `filterAndSort()` 中增加 `logo_url` 检查；重构 `fetchAveTokenLpStatus` → `fetchAveTokenDetail`，同时提取 LP 状态和 `insider_wallet_rate`；token 详情循环后增加 insider 过滤 |
| `src/server.js` | 更新前端描述文案，增加 "insider ≤50%" |
| `knowledge/pump-ranking-rules.md` | 新增 Insider 过滤规则说明 |

### 3.3 过滤流程（更新后）

```
原始候选（~470 条）
  ↓ filterAndSort: chain=solana + 市值>100K + 上线<10天 + 有图片
候选池（~22 条）     ← logo_url 过滤在此生效
  ↓ AVE token 详情: LP 状态 + insider_wallet_rate
  ↓ 排除 LP 明确未锁定 + insider > 50%
过滤后（~17 条）     ← insider 过滤排除 4-5 条
  ↓ Binance Top10 持有人占比 > 30% 排除
  ↓ 取前 20
最终榜单（17 条）
```

### 3.4 关键设计决策

- **logo_url 过滤放在 `filterAndSort`**：零 API 调用即可排除大量垃圾代币，减少后续 token 详情请求数
- **insider 阈值 0.50**：正常代币 <0.21，操控代币 >80，阈值设 0.50 留足安全余量
- **insider_wallet_rate 为 null 时不排除**：AVE 可能不返回该字段，避免误杀
- **复用 token 详情请求**：`fetchAveTokenDetail` 在同一次 API 调用中提取 LP 和 insider 数据

## 4. 测试结果

- `npm run pump-ranking`：471 条去重 → 22 候选（logo 过滤已生效）→ insider 排除 4-5 条 → 最终 17 条写入 Supabase
- `POST /api/update?type=pump`：更新 API 正常返回 17 条
- 前端验证：所有代币均有图片，无 SPACE_COIN、YELLOW_COIN、BILLIONS_NTWK_COIN 等操控代币
- 连续两次更新结果一致，过滤稳定
