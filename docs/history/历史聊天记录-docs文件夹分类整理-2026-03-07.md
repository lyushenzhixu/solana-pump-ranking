# 历史聊天记录：docs 文件夹分类整理

**日期**：2026-03-07  
**文件名**：历史聊天记录-docs文件夹分类整理-2026-03-07.md

## 1. 需求

将 `docs/` 目录下的文件按用途分类，便于查找与维护。

## 2. 分类方案

| 目录 | 说明 | 包含文件 |
|------|------|----------|
| **deploy/** | 部署与发布 | RAILWAY.md、PUSH-TO-GITHUB.md |
| **skills/** | Skill 能力与 API 总结 | README.md、AVE-CLOUD-SKILL.md、BINANCE-SKILLS.md（原有） |
| **history/** | 历史聊天记录 | 6 个「历史聊天记录-*」主题文档 |

## 3. 执行操作

1. 新建 `docs/deploy/`，将 `RAILWAY.md`、`PUSH-TO-GITHUB.md` 移入并删除根目录原文件。
2. 新建 `docs/history/`，将全部以「历史聊天记录-」开头的 md 文件移入。
3. 新增 `docs/README.md` 作为文档总索引。
4. 为 `deploy/`、`history/` 各新增 `README.md` 说明目录用途；`skills/` 已有 README。

## 4. 整理后结构

```
docs/
├── README.md
├── deploy/
│   ├── README.md
│   ├── RAILWAY.md
│   └── PUSH-TO-GITHUB.md
├── skills/
│   ├── README.md
│   ├── AVE-CLOUD-SKILL.md
│   └── BINANCE-SKILLS.md
└── history/
    ├── README.md
    └── 历史聊天记录-*.md（6 个）
```

## 5. 备注

- 若项目内仍有引用 `docs/RAILWAY.md` 或 `docs/PUSH-TO-GITHUB.md` 的链接，需改为 `docs/deploy/RAILWAY.md`、`docs/deploy/PUSH-TO-GITHUB.md`。
