## 1. 需求与实现：欢迎页入仓 + 探索按钮进榜单

**用户需求**  
- 将项目根目录的 `zhilabs欢迎页.html` 放入项目源代码，作为前端文件。  
- 点击页面「探索」按钮后进入榜单。  
- 完成后 push 到 GitHub。

**实现**  
- 在 `src/public/` 下新增前端静态入口，将欢迎页保存为 `src/public/index.html`。  
- 在欢迎页中为「探索」按钮增加点击逻辑：保留原有动效约 500ms 后跳转到 `/ranking`。  
- 在 `src/server.js` 中调整路由：  
  - `GET /`、`GET /index.html` → 读取并返回 `src/public/index.html`（欢迎页）。  
  - `GET /ranking` → 返回榜单 HTML（原内联的 `HTML_PAGE`）。  
  - `GET /api/ranking`、`GET /health` 保持不变。  
- 在 `.cursor/rules/project-conventions.mdc` 中约定：`src/public/` 为前端静态页（如欢迎页）。  
- 提交并推送到 GitHub（当时仅提交了 `src/public/index.html`、`src/server.js`、项目约定文件）。

## 2. 榜单页「返回首页」与样式

**用户反馈**  
- 部署后的榜单页面没有「返回首页」按钮。

**实现**  
- 在 `src/server.js` 的榜单 HTML 模板中增加「← 返回首页」链接，指向 `/`。  
- 为该链接增加样式类 `.back-home`，做成明显的按钮样式（背景、圆角、hover 效果），便于在榜单页顶部看到并点击。

## 3. 线上环境仍是初版单页榜单的原因与修复

**用户反馈**  
- 本地一切正常，但 Railway 通过 GitHub 部署的线上环境仍然只是初版的一页榜单（没有欢迎页、没有 /ranking、没有返回首页）。

**原因分析**  
- 在 GitHub 的 `main` 分支上，`package.json` 中仍是 `"start": "node server.js"`，即启动的是**根目录的旧 `server.js`**。  
- 根目录的旧 `server.js` 只提供榜单页，且把榜单直接放在 `/` 和 `/index.html`。  
- 新的欢迎页与榜单路由逻辑在 `src/server.js` 中，但从未被 Railway 执行，因为启动命令没有指向 `src/server.js`。  
- 因此尽管 `src/server.js` 和 `src/public/index.html` 早已推送到 GitHub，Railway 始终在跑旧入口。

**修复**  
- 将 `package.json` 中的启动脚本改为：`"start": "node src/server.js"`，并同步 `main` 等字段为 `src/index.js`。  
- 在仓库中删除根目录的旧入口文件 `server.js`、`index.js`，避免再次被错误使用。  
- 提交信息：`fix(deploy): 使用 src/server.js 启动，移除根目录旧 server.js/index.js，使 Railway 显示欢迎页与榜单`，并推送到 `main`。

**结果**  
- Railway 会使用最新提交重新部署，执行 `node src/server.js`。  
- 线上根路径为欢迎页，`/ranking` 为榜单页，榜单页顶部有「← 返回首页」按钮。

## 4. 涉及的文件与路由一览

| 路径/路由 | 说明 |
|-----------|------|
| `src/public/index.html` | 欢迎页（幻影领域），探索按钮跳转 `/ranking` |
| `src/server.js` | 主服务：`/` 欢迎页，`/ranking` 榜单页，`/api/ranking` JSON API |
| `package.json` | `"start": "node src/server.js"`，入口改为 `src` 下 |
| 根目录 `server.js` / `index.js` | 已从仓库删除，不再使用 |

## 5. 后续可做

- 若希望保留根目录的 `zhilabs欢迎页.html` 仅作本地备份，可保留不提交，或从版本库中忽略。  
- 部署后若仍看到旧页面，可在 Railway 控制台确认「Start Command」为 `npm start` 或 `node src/server.js`，并做一次 Redeploy；同时可尝试无痕或强制刷新排除缓存。
